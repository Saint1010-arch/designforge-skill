# 场景 01｜快速跑竞品 / 参考落地页 Demo

## 什么时候用 / When to use
运营、产品、市场岗经常要在会前给出"对标某家做得好的页面"的判断。老板一句"看看 X 家的落地页是怎么做的"，往往意味着你要在半天内拿出像样的拆解，而不是几张截图。

You repeatedly need a fast, defensible read of a competitor or reference landing page before a review — not just screenshots.

## 怎么用 / How to use
- 给一个 URL，进入 **Analyze 模式**。
- 工具用真实无头浏览器抓全页截图（桌面 1440 / 移动 390）、配色、字体、栅格、间距、各区块布局。
- 先返回**双语设计报告**（中文在上）：视觉语言、评分、可复用要点。

CLI: `designforge analyze https://example.com`

## 可以调什么 / What to tweak
- 只要报告：默认就先出报告，不会自动建站。
- 要落地：拿到的**同款提示词**可以直接贴进任意模型，作为后续做页面的起点。

## 你会得到 / What you get
- 一份结构化的视觉拆解（可贴进周报 / 评审文档）。
- 一段可复用的同款提示词。
- 截图与 token 证据，方便对齐和复盘。

> 现实补充：老板的对标对象经常临时变。把"拆解"做成一条命令，比每次手动截图标注省一整个下午。
