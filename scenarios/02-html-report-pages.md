# 场景 02｜批量产出 HTML 报告 / 案例 / 展示页

## 什么时候用 / When to use
运营和 PM 经常要把数据、案例、复盘做成"能直接发出去"的单文件 HTML —— 不依赖搭建工具，发给老板或客户打开就能看，还要好看、统一风格。

You need polished, send-ready single-file HTML reports / case pages / showcases, fast and on-brand.

## 怎么用 / How to use
- **HTML 模式**：输入一个链接、或上传一个已有的 HTML 文件。
- 工具先给报告（结构、视觉、可改进点），再按要求**重做 / 升级**成更专业的单文件 HTML。
- 支持多个文件的**分析、对比、融合**，把零散的几页统一成一套视觉。

详见 `references/html-forge.md`。

## 可以调什么 / What to tweak
- 升级方向：更克制的商务风 / 更有设计感的展示风 / 贴近某个品牌。
- 自定义需求：换标题文案、配色主色、加入你自己的提示词与约束（自由度高，按需叠加）。
- 输出：单文件 HTML（内联样式，发出去即可打开）。

## 你会得到 / What you get
- 先一份 HTML 体检报告，再一个升级后的单文件页面。
- 多文件融合时：一套统一风格的成品 + 说明。

> 现实补充：需求多变时，先出报告再动手，能避免"做完才发现方向不对"的返工。
