# 场景 05｜对一批页面做快速、结构化的视觉评审

## 什么时候用 / When to use
手上有一批页面（自家的、竞品的、历史版本的），需要快速、统一口径地评一遍，而不是凭感觉说"这个不好看"。

You need a fast, consistent, structured visual review across several pages — not vibes.

## 怎么用 / How to use
- 对每个 URL/HTML 跑 **Analyze**，拿到统一字段的报告（配色、字体、布局、动效、评分）。
- 多个 HTML 文件可直接**对比**，输出差异与建议。

## 可以调什么 / What to tweak
- 评审维度：是否额外关注可访问性、信息层级、移动端适配。
- 输出形式：逐页报告，或汇总成一页对比。

## 你会得到 / What you get
- 同一套字段的可比报告，结论可以直接进评审文档。

> 现实补充：统一口径的评审，比每个人各说各话更容易让老板拍板。
