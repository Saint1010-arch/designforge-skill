# 场景手册 / Scenarios

这些是把 DesignForge 用在真实工作里的几条"打法"。每条都说明：什么时候用、可以调什么、你会得到什么。它们都遵循同一套原则——**先出报告，再决定要不要建页面**；做的是**同款风格**，不是 1:1 复制。

These are concrete, work-grade playbooks. Each says when to use it, what to tweak, and what you get. All follow the same rule: **report first, build only if asked**; recreate the *style*, never a 1:1 copy.

- `01-competitor-demos.md` — 快速跑竞品 / 参考落地页 Demo
- `02-html-report-pages.md` — 批量产出 HTML 报告 / 案例 / 展示页
- `03-brand-design-language.md` — 对齐品牌设计语言 + 同款提示词
- `04-fusion-directions.md` — 两个方向按权重融合
- `05-fast-visual-review.md` — 一批页面的快速结构化评审

> 背景：在运营 / 产品节奏里，需求经常临时变、老板想"多看几个版本再挑"。这些场景的共同目的，是把重复的视觉拆解、产出、对齐变成几条可复用的动作。
